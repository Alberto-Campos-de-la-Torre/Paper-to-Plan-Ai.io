El Problema: Documentos Clinicos en Papel

Los profesionales de la salud generan diariamente notas clinicas, recetas y resultados de laboratorio en papel. Digitalizarlos manualmente consume tiempo valioso y subir datos de pacientes a la nube compromete la privacidad.

  • OCR tradicional falla con letra medica manuscrita.
  • Datos de pacientes en APIs en la nube = riesgo de privacidad.
  • Falta de estructura: sin formato SOAP, sin codigos CIE-10 automaticos.

La Solucion: MEGI Records

Vision Multimodal

Interpreta documentos manuscritos e impresos con modelos de vision (qwen3-vl).

Analisis SOAP + CIE-10

Estructura automatica en Subjetivo, Objetivo, Evaluacion y Plan con codigos diagnosticos.

100% Local con Ollama

Los datos clinicos nunca salen de tu equipo. Sin nube, sin riesgos.

Companion Movil

Escanea QR y usa tu celular como escaner de documentos medicos.

Funciones Principales

Todo lo que necesita un consultorio para digitalizar y gestionar expedientes clinicos.

Gestion de Pacientes

CRUD completo: datos demograficos, alergias, condiciones, tipo de sangre, contactos de emergencia.

Tablero Kanban

Estadisticas

Multi-Usuario

Tiempo Real

Flujo de Trabajo

Del papel al expediente digital en 7 pasos automatizados

1

Captura

Sube imagen, usa webcam, companion movil o escribe manualmente una consulta.

2

OCR

Extraccion de texto del documento fisico via EasyOCR + vision multimodal.

3

Clasificacion

La IA identifica el tipo: consulta, receta, resultado de laboratorio o referencia.

4

Analisis SOAP

Estructuracion en Subjetivo / Objetivo / Evaluacion / Plan con codigos CIE-10 y score de confianza.

5

Extraccion

Recetas (medicamento, dosis, frecuencia) y valores de laboratorio se guardan en tablas dedicadas.

6

Revision

El doctor revisa el analisis, corrige si es necesario y marca como revisado.

7

Exportacion

Genera PDFs profesionales de notas medicas y recetas listos para imprimir.

Esquema de Datos

Base de datos SQLite local (megirecords.db)

patients

Nombre, nacimiento, genero, tipo de sangre, alergias (JSON), condiciones (JSON), codigos CIE-10, contactos.

consultations

Paciente, doctor, tipo de documento, texto raw, analisis SOAP (JSON), estado, imagen, prioridad.

prescriptions

Medicamento, dosis, frecuencia, duracion, instrucciones. Vinculado a consulta y paciente.

lab_results

Estudio, valor, unidad, rango de referencia, indicador de anomalia, fecha.

users

Doctores y staff con username y PIN de autenticacion.

corrections

Historial de correcciones OCR para mejorar la precision con el tiempo.

Instalacion

# Clonar y cambiar a branch MEGI Records

git clone https://github.com/Alberto-Campos-de-la-Torre/Paper-to-Plan-Ai.io.git

cd Paper-to-Plan-Ai.io

git checkout feat/megi-records


# Backend

python -m venv venv && source venv/bin/activate

pip install -r requirements.txt

python backend/server.py


# Frontend

cd desktop-app && npm install && npm run dev


# Desktop (Tauri)

npm run tauri dev


# Modelos de IA (requiere Ollama instalado)

ollama pull gemma3

ollama pull qwen3-vl:8b
